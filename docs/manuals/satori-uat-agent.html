Satori UAT Agent

Comprehensive User Acceptance Testing automation for legal document processing systems

Overview

The Satori UAT Agent is an intelligent testing automation system designed specifically for the Tiger-Monkey legal document processing platform. It combines advanced browser automation with intelligent test case generation to ensure comprehensive quality assurance for front-end features.

Built on Playwright browser automation technology, the UAT Agent provides automated test execution, intelligent test case generation from development tasks, and comprehensive reporting. It bridges the gap between development requirements and quality assurance by automatically translating acceptance criteria into executable test cases.

The system intelligently analyzes development task files, extracts front-end requirements, and generates structured UAT test cases that validate user-facing functionality. This ensures that every front-end feature is thoroughly tested before deployment to production.

Core Features

Development Task Analysis

Automatically reads development task files, extracts requirements, and generates corresponding UAT test cases with mapped success criteria.

Intelligent Input Processing

Detects input type (dev task file, UAT test file, or description string) and responds with appropriate analysis or execution.

Browser Automation

Full Playwright integration for automated navigation, interaction, screenshot capture, and comprehensive UI validation.

Structured Reporting

Generates detailed test reports with pass/fail analysis, visual documentation, and actionable feedback for developers.

Voice Notifications

Real-time audio alerts for test completion, failures, status updates, and user intervention requests during testing.

Acceptance Criteria Mapping

Automatically converts development acceptance criteria into executable success criteria for comprehensive feature validation.

Visual Documentation

Automatic screenshot capture for initial states, test failures, successes, and comprehensive visual test evidence.

Legal Domain Expertise

Specialized for legal document processing workflows, compliance requirements, and court-ready document validation.

System Architecture

Component Overview

  • Input Processor: Detects and routes different input types (dev tasks, UAT tests, descriptions)
  • Task Analyzer: Evaluates development tasks for front-end testing requirements and extracts acceptance criteria
  • Test Generator: Creates structured UAT test cases with mapped success criteria and test interactions
  • Browser Engine: Playwright-powered automation for web interaction, navigation, and validation
  • Criteria Mapper: Converts development acceptance criteria into executable UAT success criteria
  • Reporting System: Generates comprehensive test documentation, screenshots, and results analysis
  • Notification System: Audio and visual alerts for test status updates and user intervention requests

Integration Points

  • Tiger Service: Document processing and data extraction validation
  • Monkey Service: Document generation and template rendering testing
  • Web Dashboard: Case management interface and user workflow testing
  • File System: Test artifact storage and organization

Development Task Analysis

Intelligent Requirements Extraction

The UAT Agent automatically analyzes development task files to identify front-end components and extract testable requirements. It converts acceptance criteria into executable success criteria and generates comprehensive test cases.

1. Parse Requirements

Extracts functional requirements, technical specifications, and acceptance criteria from development tasks.

2. Map Criteria

Converts acceptance criteria into measurable success criteria with specific validation points.

3. Generate Tests

Creates structured UAT test files with interactions, expected outcomes, and validation steps.

Analysis Decision Matrix

Task Type Front-end Impact UAT Action Example

Usage Methods

1. Development Task File Analysis

Analyze a development task file and automatically generate corresponding UAT test cases.

/uat TM/tasks/example_dev_task.md

# Output: Generates TM/tasks/UAT/uat_footer_1.md with:
# - Success criteria based on acceptance criteria
# - Test interactions for all footer functionality  
# - Expected outcomes for each requirement

2. Task Description Analysis

Provide a development task description and the UAT Agent will analyze it for front-end testing requirements.

/uat "Add new case creation form with file upload functionality"

3. Test Execution Mode

Execute existing UAT test cases by providing the test instruction file path.

/uat TM/tasks/UAT/uat_dashboard_1.md

4. Interactive Testing Mode

Step-by-step testing with manual validation and real-time decision making.

/uat --interactive TM/tasks/UAT/uat_form_validation.md

# Interactive Features:
# - Pause between each test step for manual inspection
# - Voice prompts for user decisions and confirmations
# - Dynamic test modification during execution
# - Manual screenshot triggering at any point
# - Failure recovery options (retry, skip, modify)

Note: Interactive mode is currently under development. It will enable exploratory testing and debugging scenarios where step-by-step validation is preferred over automated execution.

Test Case Structure

Standard Test File Format

# UAT [Feature Name] Task [Number]

## APP: Satori Legal Case Management System

## APP URL:
http://127.0.0.1:8000/

## UAT DIR:
TM/tasks/UAT/
 - tasks to work on marked with uat_ prefix
 - results saved in the same directory

## Development Context
[Brief description of the development task]

## Steps
1. Navigate to the target page/feature
2. Take initial screenshot
3. Execute test interactions
4. Validate against success criteria
5. Generate test report

## Success Criteria
1. Page loads without errors
2. All UI elements are present
3. Functionality works as expected
4. Data displays correctly
5. User interactions work properly

## Test Interactions
[Specific user actions to test]

## Expected Outcomes
[What should happen after each interaction]

Best Practices

Test Design

  • Create specific, measurable success criteria
  • Include both positive and negative test cases
  • Test edge cases and error conditions
  • Validate accessibility and responsive design
  • Test keyboard navigation and screen readers

Test Execution

  • Always take initial screenshots before interactions
  • Use descriptive element names for better readability
  • Wait for elements to be ready before interaction
  • Clean up browser resources after testing
  • Use absolute paths for file operations

Documentation

  • Generate comprehensive test reports
  • Include screenshots for visual validation
  • Document all test interactions and outcomes
  • Maintain version control for test cases
  • Update test cases when requirements change

Troubleshooting

Common Issues

Browser Connection Failed

The browser automation cannot connect to the application.

Solution: Ensure the application is running on the specified URL (typically http://127.0.0.1:8000/)

Element Not Found

The test cannot locate specific UI elements.

Solution: Take a fresh snapshot to get updated element references. Check if the UI has changed.

Test Timeout

Tests are timing out during execution.

Solution: Add wait conditions for dynamic content. Increase timeout values for slow-loading pages.

Screenshot Issues

Screenshots are not capturing correctly.

Solution: Check file permissions in the UAT directory. Ensure the browser window is properly sized.

Debugging Tips

  • Use browser console messages to identify JavaScript errors
  • Check network requests for failed API calls
  • Verify element selectors with browser developer tools
  • Test manually before automating complex interactions
  • Review generated test reports for detailed failure information
