Welcome to the Satori Tiger-Monkey System

This system is a powerful, dual-service pipeline designed to streamline the processing of legal documents for consumer protection law cases. It consists of two main services: Tiger and Monkey.

üêÖ Tiger Service

Tiger is the extraction and analysis engine. It takes raw legal documents (PDFs, DOCX, etc.) as input and uses advanced OCR and natural language processing to extract key legal entities, facts, and case information. The output is a highly structured, consolidated JSON file.

üêí Monkey Service

Monkey is the document generation engine. It consumes the JSON file produced by Tiger and uses a sophisticated templating system to generate court-ready legal documents in HTML format. These documents can then be printed to PDF for filing.

üêÖ Tiger: Data Extraction and Consolidation

The Tiger service is responsible for processing raw legal documents (PDFs, DOCX, etc.) from a case folder, extracting key information, and consolidating it into a single structured JSON file.

Command

To run the Tiger service on a case directory, use the `hydrated-json` command:

/Users/corelogic/satori-dev/TM/tiger/run.sh hydrated-json <path_to_case_directory> -o <path_to_output_directory>

Example

This example processes the `youssef` case and saves the consolidated JSON file to the `outputs/tests` directory.

/Users/corelogic/satori-dev/TM/tiger/run.sh hydrated-json /Users/corelogic/satori-dev/TM/test-data/cases/youssef -o /Users/corelogic/satori-dev/TM/outputs/tests/

Expected Output

Tiger will create a new JSON file in the specified output directory. The filename will be based on the case name and date, for example:

/Users/corelogic/satori-dev/TM/outputs/tests/hydrated_FCRA_Youssef_Eman_20250704.json

üêí Monkey: Document Generation

The Monkey service takes the structured JSON file generated by Tiger and uses it to create formatted HTML documents.

Generate Case Review Document

Use the `review` command to generate a `case_review.html` file.

/Users/corelogic/satori-dev/TM/monkey/run.sh review <path_to_generated.json> -o <path_to_output_review.html>

Example

/Users/corelogic/satori-dev/TM/monkey/run.sh review /Users/corelogic/satori-dev/TM/outputs/tests/hydrated_FCRA_Youssef_Eman_20250704.json -o /Users/corelogic/satori-dev/TM/outputs/tests/review_youssef.html

Generate Complaint Document

Use the `build-complaint` command with a specific template to generate the complaint document.

/Users/corelogic/satori-dev/TM/monkey/run.sh build-complaint <path_to_generated.json> --template <template_name.html> -o <path_to_output_directory>

Example

/Users/corelogic/satori-dev/TM/monkey/run.sh build-complaint /Users/corelogic/satori-dev/TM/outputs/tests/hydrated_FCRA_Youssef_Eman_20250704.json --template ground_truth_complaint.html -o /Users/corelogic/satori-dev/TM/outputs/tests/

This will create a `complaint.html` file inside a dated subfolder within the `processed` directory of your output path.

üî¨ End-to-End Testing

This section provides a step-by-step guide to running a full end-to-end test of the Tiger-Monkey pipeline using the `youssef` case as an example.

Step 1: Run Tiger

Generate the consolidated JSON file from the `youssef` case files.

/Users/corelogic/satori-dev/TM/tiger/run.sh hydrated-json /Users/corelogic/satori-dev/TM/test-data/cases/youssef -o /Users/corelogic/satori-dev/TM/outputs/tests/

Step 2: Run Monkey (Review)

Generate the case review document.

/Users/corelogic/satori-dev/TM/monkey/run.sh review /Users/corelogic/satori-dev/TM/outputs/tests/hydrated_FCRA_Youssef_Eman_20250704.json -o /Users/corelogic/satori-dev/TM/outputs/tests/review_youssef.html

Step 3: Run Monkey (Complaint)

Generate the complaint document.

/Users/corelogic/satori-dev/TM/monkey/run.sh build-complaint /Users/corelogic/satori-dev/TM/outputs/tests/hydrated_FCRA_Youssef_Eman_20250704.json --template ground_truth_complaint.html -o /Users/corelogic/satori-dev/TM/outputs/tests/

Step 4: Verify

Check the `outputs/tests` directory for the generated `review_youssef.html` and `complaint.html` files.
